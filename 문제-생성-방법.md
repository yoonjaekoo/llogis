# 문제 생성 방법 (Template-based Problem Generation)

## 개요

Logis는 **템플릿 기반 문제 생성 엔진**을 사용합니다. 31개의 수학 문제 템플릿이 `backend/data/templates.json`에 정의되어 있으며, 각 템플릿은 변수, 제약조건, 문제 템플릿, 정답 공식으로 구성됩니다.

생성 엔진은 `backend/src/generation/` 디렉토리에 위치하며, 순수 함수형 파이프라인으로 동작합니다. `eval()`을 사용하지 않는 안전한 Recursive-descent 파서를 포함합니다.

---

## 생성 과정

```
generateProblem(template):
  1. 변수 생성: 템플릿의 variables 정의에 따라 랜덤 값 생성
  2. 제약조건 검증: 모든 constraints를 통과하는지 확인
     - 실패 시 1로 돌아가 최대 50회 재시도
  3. 문제 렌더링: problem_template의 {{expression}}을 계산된 값으로 치환
  4. 정답 계산: answer_formula를 평가하여 정답 도출
  5. 정답 포맷팅: 소수점 정리, boolean/배열 변환
```

---

## 템플릿 형식

```json
{
  "id": "MS-EQ1-001",
  "unit": "일차방정식",
  "title": "연속하는 세 자연수의 합",
  "difficulty": 12000,
  "variables": {
    "S": { "type": "integer", "min": 30, "max": 300 }
  },
  "constraints": ["S % 3 == 0"],
  "problem_template": "연속하는 세 자연수의 합이 {{S}}일 때, 이 중 가장 큰 수를 구하시오.",
  "answer_formula": { "type": "expression", "value": "(S / 3) + 1" },
  "concepts": ["일차방정식의 활용", "연속하는 수"]
}
```

### 필드 설명

| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | string | 고유 식별자. 예: `MS-EQ1-001` |
| `unit` | string (optional) | 단원 분류. 예: `"일차방정식"` |
| `title` | string | 사람이 읽을 수 있는 제목 |
| `difficulty` | number | 난이도 점수 (10000~20000) |
| `reward_rating` | number (optional) | 문제 해결 시 획득 레이팅. 없으면 difficulty 기준 자동 계산 |
| `variables` | object | 변수 이름 → 변수 정의 맵 |
| `constraints` | string[] | 논리 표현식 배열 (모두 `true`여야 통과) |
| `problem_template` | string | `{{표현식}}`으로 변수/계산 결과를 치환하는 템플릿 |
| `answer_formula` | object | `{ "type": "expression", "value": "..." }` 형태 |
| `concepts` | string[] (optional) | 개념 태그 배열 |

---

## 변수 타입

### integer

`min` 이상 `max` 이하의 정수를 균등 확률로 생성합니다.

```json
{ "type": "integer", "min": 30, "max": 300 }
{ "type": "integer", "min": -10, "max": -2 }
```

### float

`min` 이상 `max` 미만의 실수를 생성합니다. (현재 템플릿에서 사용 중인 예시 없음)

```json
{ "type": "float", "min": 0.0, "max": 1.0 }
```

### choice

`choices` 배열에서 무작위로 하나를 선택합니다. 숫자뿐 아니라 문자열도 가능하나, 내부적으로 `Number()` 변환을 거칩니다.

```json
{ "type": "choice", "choices": [2, 3, 4, 5] }
```

### boolean

`true` 또는 `false`를 50% 확률로 생성합니다.

```json
{ "type": "boolean" }
```

---

## 제약조건 (Constraints)

제약조건은 문자열 배열로, 각 문자열은 `true` 또는 `false`로 평가되는 표현식입니다. **모든** 제약조건이 `true`여야 생성에 성공합니다.

제약조건이 까다로울수록 재시도 횟수(최대 50회)를 초과하여 생성에 실패할 수 있으므로, 충분한 확률(10% 이상)이 보장되도록 설계해야 합니다.

### 제약조건 예시

| 표현식 | 의미 |
|--------|------|
| `S % 3 == 0` | S가 3의 배수 |
| `age_diff % (multiplier - 1) == 0` | age_diff가 (multiplier-1)로 나누어떨어짐 |
| `a > 0` | a가 양수 |
| `b < 0` | b가 음수 |
| `a != b` | a와 b가 다름 |
| `(sum + diff) % 2 == 0` | sum + diff가 짝수 |
| `x < y` | x가 y보다 작음 |
| `180 - a - b > 0` | 삼각형 각도의 합이 180 미만 |
| `cost % 10 == 0` | cost가 10의 배수 |
| `(a + b + c + d + e) % 5 == 0` | 다섯 수의 합이 5의 배수 |
| `n == 10 \|\| n == 11 \|\| n == 12` | n이 10, 11, 12 중 하나 |
| `target_conc > initial_conc` | target이 initial보다 큼 |
| `days_a * multiplier % (1 + multiplier) == 0` | 작업일 조건 |

## 템플릿 렌더링 (`{{expression}}`)

`problem_template`에서 `{{...}}`로 감싸진 부분은 평가되어 값으로 치환됩니다.

### 예시

| 템플릿 | 변수 | 렌더링 결과 |
|--------|------|-------------|
| `합이 {{S}}` | `{S: 60}` | `합이 60` |
| `{{initial_conc}}\\%의 소금물` | `{initial_conc: 8}` | `8\\%의 소금물` |
| `$\|b\| = {{-b}}$` | `{b: -5}` | `$\|b\| = 5$` |
| `${{a}}x + {{b}} = {{c}}$` | `{a:2, b:3, c:11}` | `$2x + 3 = 11$` |
| `총 $\frac{{{dist * (speed1 + speed2)}}}{{{speed1 * speed2}}}$` | `{dist:20, speed1:4, speed2:3}` | `총 $\frac{140}{12}$` |
| `${{a}}x^2 - {{a*a}}x$` | `{a:3}` | `$3x^2 - 9x$` |

**중요:** `{{...}}` 안에는 단순 변수명뿐 아니라 **임의의 수식**이 들어갈 수 있습니다. 위 예시 중 `{{a*a}}`, `{{dist * (speed1 + speed2)}}` 등이 이에 해당합니다.

---

## 정답 공식 (answer_formula)

항상 `{ "type": "expression", "value": "<표현식>" }` 형태입니다.

`value`에 있는 표현식은 생성된 변수들을 사용하여 평가되며, 결과는 다음 규칙에 따라 포매팅됩니다:

- **boolean**: `true` → `1`, `false` → `0`
- **number[]**: 각 요소를 10자리에서 반올림 후 `', '`로 결합
- **number (정수)**: 그대로 반환
- **number (실수)**: `Math.round(value * 1e10) / 1e10` 처리 후 반환

---

## 레이팅 보상 (reward_rating)

`templates.json`에 `reward_rating`이 명시되지 않은 경우, 템플릿 로드 시 **전체 템플릿 중 difficulty 순위**에 따라 자동 계산됩니다:

- **하위 1/3**: 5,000 ~ 10,000
- **중간 1/3**: 10,000 ~ 14,000
- **상위 1/3**: 15,000 ~ 20,000

```json
{
  "templateIds": ["MS-EQ1-001", "MS-EQ1-002"],   // 특정 템플릿 ID 목록
  "count": 10                                      // 생성할 문제 수 (1~50)
```

```json
{
  "unit": "일차방정식",
  "count": 5
}
```

```json
{
  "concept": "인수분해",
  "count": 3
}
```

아무 필터 없이 요청하면 전체 템플릿에서 랜덤 선택:

```json
{
  "count": 10
}
```

### `POST /api/problems/templates/reload`

`templates.json`을 다시 로드합니다. (인증 필요, admin 권한 필요)

## 새 템플릿 추가 방법

`backend/data/templates.json`에 새 객체를 추가합니다. 다음 규칙을 따라야 합니다:

1. **`id`**는 기존과 중복되지 않는 고유값 사용
2. **`variables`**는 생성 엔진이 지원하는 타입만 사용 (`integer`, `float`, `choice`, `boolean`)
3. **`constraints`**는 너무 까다롭지 않게 설계 (최소 5% 이상의 확률 보장 권장)
4. **`problem_template`**에서 `$...$`나 `$$...$$`는 KaTeX LaTeX 문법으로 처리됨
5. **`answer_formula.value`**는 `{{expression}}` 없이 순수 수식만 작성
